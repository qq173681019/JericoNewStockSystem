# ⚠️ 常见问题：找不到 run_web_ui.py 文件

## 问题描述
运行 `启动Web界面.bat` 时出现错误：
```
python: can't open file '...\run_web_ui.py': [Errno 2] No such file or directory
```

## 原因
您的本地代码不是最新版本。`run_web_ui.py` 文件是在最近的提交中添加的，您需要从GitHub拉取最新代码。

## 解决方案

### 方法1：使用Git命令行（推荐）

打开 **Git Bash** 或 **命令提示符**，然后：

```bash
# 进入项目目录
cd C:\Users\你的用户名\Documents\GitHub\JericoNewStockSystem

# 拉取最新代码
git pull origin copilot/build-ui-with-pencil-plugin-again

# 或者先获取更新再切换分支
git fetch
git checkout copilot/build-ui-with-pencil-plugin-again
```

### 方法2：使用GitHub Desktop

1. 打开 **GitHub Desktop**
2. 选择 **JericoNewStockSystem** 仓库
3. 点击顶部的 **Fetch origin**
4. 在左侧分支列表中，选择 **copilot/build-ui-with-pencil-plugin-again**
5. 点击 **Pull origin**

### 方法3：在VS Code中

1. 打开 VS Code
2. 按 `Ctrl+Shift+P` 打开命令面板
3. 输入 "Git: Pull" 并选择
4. 确保当前分支是 `copilot/build-ui-with-pencil-plugin-again`

## 验证更新

更新后，检查以下文件是否存在：

```
JericoNewStockSystem/
├── run_web_ui.py              ← 必须存在
├── 启动Web界面.bat             ← 必须存在
├── START_WEB_UI.bat           ← 必须存在
├── 如何启动.md                 ← 必须存在
└── web_ui/                    ← 必须存在
    ├── static/
    ├── templates/
    └── README.md
```

## 更新后重新运行

1. 双击 **`启动Web界面.bat`**
2. 如果提示缺少依赖，批处理文件会自动安装
3. 浏览器会自动打开 http://127.0.0.1:5000

## 其他常见问题

### 桌面版报错 "ModuleNotFoundError: No module named 'dotenv'"

**解决方案**：
```bash
pip install python-dotenv
# 或安装所有依赖
pip install -r requirements.txt
```

### 无法安装依赖包

**解决方案**：
```bash
# 升级pip
python -m pip install --upgrade pip

# 然后重试
pip install Flask Flask-CORS python-dotenv
```

### 还是无法运行？

请确保：
1. ✅ 已安装 Python 3.8+
2. ✅ Python 已添加到系统 PATH
3. ✅ 已拉取最新代码
4. ✅ 在正确的项目目录中

如果问题仍然存在，请在GitHub Issue中附上完整的错误信息。
